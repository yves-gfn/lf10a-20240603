CSS-Mitschrift

Was ist CSS?

CSS steht für Cascading Stylesheet. CSS kann man sich vorstellen, wie eine Formatvorlage aus z.B. Word. Wir können unterschiedliche Einstellungen für unterschiedliche Elemente vornehmen, die dann meistens auf allen Seiten angewendet werden.

Was kann man z.B. mit CSS verändern?

  • Schriftart, Schriftfarbe, Schriftgröße usw...
  • Hintergrundfarbe, Hintergrundbilder und vieles mehr

Mit CSS kann ebenfalls für die gleiche Webseite unterschiedliche Geräte ansprechen, z.B. Drucker, Smartphones, Tablets... Ebenfalls werden mit CSS Webseiten responsiv gemacht. Das bedeutet, dass der Inhalt und das Layout sich automatisch auf die Ausgabegeräte-Breite anpasst.
Mit CSS gelingt uns die Trennung zwischen Form und Struktur. In HTML sollten möglichst wenig Styling-Informationen übrig bleiben, dafür wird zukünftig CSS verwendet. Die aktuelle von CSS ist CSS3, die kam 2010 raus. Die erste Version kam 1996 raus.

Wie verwendet man CSS?

  • Man kann CSS als eigene Datei erstellen und dann in HTML per Link-Tag einbinden
  • Man kann CSS in den Head-Bereich mit dem style-Tag schreiben
  • Man kann CSS direkt "inline" verwenden, das heißt direkt beim Tag. Hierfür nutzt man das "style-Attribut".

Beispieleinbindung im head

Der <style>-Bereich definiert die Stelle, indem die CSS-Vorgaben gemacht werden. Der MIME-Type (wie bereits gelernt) muss angegeben werden mit type="text/css". Man kann unterschiedliche media - Anweisungen geben, z.B.

  • media = "screen" (Bildschirmdarstellung)
  • media = "print" (Druckdarstellung)
  • media = "all" (deckt alle Medien ab)

CSS arbeitet nach dem Vererbungsprinzip

Viele CSS Vorgaben gelten ebenfalls für verschachtelte Elemente in einem Element, werden als "weiter vererbt." Es gibt aber ebenefalls einige Eigenschaften, die nicht vererbt werden. Nachschauen kann man hier: CSS vererbbare Eigenschaften

Vererbungskonzept

  • Übergeordnete Elemente werden als Ancestor-Elemente (Vorfahren) definiert
  • Das direkt übergeordnete Element wird als Parent-Element (Eltern) bezeichnet
  • In einem Element enthaltene Tags nennt Descendants (Nachfahren)
  • Das direkt enthaltene Element ist das Child-Element (Kinder)

Vertiefend zu em und rem

Em und rem sind wichtig, weil wir feste pixel-größen vermeiden wollen. Die Gründe z.B. Resonsives Design: Im Gegensatz gedruckten Seiten müssen Weblayouts an unterschiedliche Anzeigengrößen angepasst sein. Eine Schriftgröße, die bei euch auf dem Desktop gut lesbar ist, muss nicht auf dem Smartphone gut lesbar sein, sondern wird z.B. viel zu klein angezeigt. Ebenfalls ist die Barrierefreiheit wichtig, die Probleme haben kann mit festen - Pixelwerten. Bitte zukünftig bei Texten immer em oder rem verwenden!

Positionierung von Elementen

Mit CSS kann jedes Element beliebig platziert werden. Mit der Eigenschaft position kann angegeben werden, welche Positionierungsmethode verwendet wird.

Eigenschaft Beschreibung
position: static Standardplatzierungsmethode, Elemente werden platziert, indem sie sich gegenseitig verdrängen dem Quellcode entsprechend
position: relative Ähnlich wie static, legt Elemente anhand des Objekflusses ab, lässt sich aber mit top, left u.a. manipulieren
position: absolute Verhält sich komplett anders wie static und relative. Das Element wird aus dem Objektfluss herausgenommen und auf eine höhere Ebene gesetzt. Diese Ebene kann bei mehreren Elementen mit dem z-index definiert werden. position: absolute orientiert sich bei seiner Positionierung an das Element im Objektfluss was nicht position: static ist. Damit kann es gut mit position:relative verwenden, als eine Art "ankerpunkt", an dem sich position:absolute orientiert.
position: fixed Funktioniert ähnlich wie position:absolute bleibt an Ort und Stelle und scrollt mit und über den restlichen Inhalt hinweg.

Ändern der Anzeigeart

Wie bereits bekannt gibt es in HTML zwei Arten von Elementen. Blockelemente (z.B. div, p, h1-h6, header, main, footer, nav) die Absätze erzeugen. Des Weiteren auch inline-Elemente (z.B. strong, em, b, i, a, span), die innerhalb eines Fließtextes benutzt werden können. Mit CSS kann die Darstellungsart über die display-Eigenschaft verändert werden.

Eigenschaft Beschreibung
display: block Wandelt inline-Element in ein Block-Element um, Element erzeugt neue Zeile und nimmt unangepasst die gesamte ihm zur Verfügung stehende Breite ein, Breite und Höhe können definiert werden
display: inline Wandelt ein Blockelement in ein Inline-Element um, Element wird im laufenden Textfluss angezeigt, Breite und Höhe können nicht definiert werden
display: inline-block Erzeugt einen Block, für den Breite und Höhe angegeben werden können, lässt Elemen jedoch im laufenden Textfluss, ähnlich dem img-Element
display: none Das Element wird nicht angezeigt und nimmt auch keinen Raum mehr ein (sehr nützlich für evtl. Responsive, wenn gewisse Elemente bei Mobile nicht angezeigt werden sollen)

nth-child und nth-of-type Auswahl

Für das Durchzählen von Elementen gibt es zwei Pseudo-Klassen und zwar nth-child und nth-of-type. Diese erwarten in der runden Klammer eine Angabe, welche Elemente ausgwählt werden.

nth-child(3) - direkte Auswahl

nth-child(odd) - ungerade Auswahl

nth-child(evene) - gerade Auswahl

nth-child(3n)

Mit der Formel 3n wird jedes 3te Element gewählt. Auch ein Offset kann angebenen werden, z.B 2n +1. Hier wird der Start der Zählung um 1 verändert.

nth-of-type(odd)

nth-of-type funktioniert genau so wie nth-child, nur dass nicht jedes Nachfahr-Element "durchgezählt", sondern nur der angegebene Typ. Es können die gleichen Angaben wie in nth-child verwenden werden, z.B. odd, even, eine genaue Zahl oder Xn + Offset.

first-child, last-child und only-child

Mit den Pseudo-Klassen first-child, last-child und only-child können entweder das erste Element, das letzte oder das einzige Element ausgewählt werden. Hierbei wird die Bedingung geprüft, ob das erste tatsächlich vom entsprechenden Typ ist, siehe Beispiel. Last-Child funktioniert genau so wie first-child, nur eben das letzte Element. Bei only-child darf nur ein Element in der Liste der Bedingung entsprechen, dann wird es formatiert.
